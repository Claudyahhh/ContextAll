# GitHub checklist

[Documentation index](README.md) · [Privacy](PRIVACY.md)

## Before the first push

- [ ] Choose repository name and one-sentence description.
- [ ] Confirm all examples are fictional.
- [ ] Search for names, emails, phone numbers, home paths, and private URLs.
- [ ] Confirm English and Chinese README links point to each other.
- [ ] Confirm documentation links resolve from their own directories.
- [ ] Confirm index templates match `document-schema.md`.
- [ ] Choose and add a license deliberately.
- [ ] Add screenshots only after the interface exists.
- [ ] Validate the nested skill.
- [ ] Test installation in a clean temporary vault.
- [ ] Confirm README links resolve.

## Later release checks

- add a 60-second demo GIF after a UI exists;
- add a compatibility table when more agents are tested;
- add roadmap and contribution rules before accepting contributions;
- keep examples fictional as the test fixture library grows;
- audit design examples so they never become implied user defaults.
