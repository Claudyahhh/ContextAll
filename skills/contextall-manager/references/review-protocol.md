# Review protocol

## Per-write checks

- Source metadata and privacy are present.
- Meaningful original wording and attribution remain.
- Moved fragments remain traceable when order matters.
- Source and Derived Context are visibly separated.
- Topic navigation maps to real headings.
- `topic_index` paths and fragment IDs resolve.
- Source, entity, topic, timeline, and total indexes were updated when required.
- Downstream routes point back to Source Context.

## Weekly review

- Process inbox captures.
- Repair missing source references and incomplete fragment grouping.
- Normalize new aliases.
- Replace vague topic entries with concrete object + judgment/action/tension.
- Review Derived Context without treating it as fact.

## Monthly review

- Merge only genuinely duplicate source records.
- Promote mature material to destinations.
- Sample-test retrieval through source, entity, topic, and time views.
- Update profile, privacy boundaries, and user-owned design profile.
- Archive stale interpretations while retaining Source Context.

## Failure patterns

| Failure | Repair |
|---|---|
| summary replaced the original | restore Source Context and move summary to derivation |
| scattered remarks lost their order | add fragment IDs and source reference |
| everything lands in one folder | classify by primary meaning |
| filenames are the only index | add vault and section-level indexes |
| topic index says “work” or “AI” | use a concrete object plus judgment/action/tension |
| aliases fragment one entity | choose a canonical name and map aliases |
| AI interpretation looks like user belief | add layer label and provenance notice |
| example colors become user defaults | request and confirm a personal design profile |
| public repo contains personal data | remove from history and use fictional fixtures |
