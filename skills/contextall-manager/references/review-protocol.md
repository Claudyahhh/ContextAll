# Review protocol

## Per-write checks

- Confirm the canonical source was preserved.
- Confirm edits were structural, not stylistic.
- Confirm uncertain speaker attribution is marked.
- Confirm AI additions are isolated.
- Confirm index links and heading paths resolve.
- Confirm privacy level and downstream route.

## Weekly review

- Process inbox notes.
- Repair broken navigation and aliases.
- Note recurring entities that deserve an index entry.
- Review newly generated derivations without treating them as facts.

## Monthly review

- Merge only genuinely duplicate notes.
- Promote mature ideas to destination projects.
- Update profile, privacy boundaries, and design tokens.
- Archive stale derivations while retaining source evidence.
- Sample-check retrieval: entity → note → exact section.

## Failure patterns

| Failure | Repair |
|---|---|
| Summary replaced the original | Restore source and move summary to derivation |
| Everything lands in one folder | Reclassify by primary meaning |
| Many duplicate notes | Strengthen entity aliases and append criteria |
| Notes are searchable only by filename | Add entities and topic index |
| AI interpretation looks like user belief | Add a hard divider and provenance label |
| Public repo contains personal data | Remove it from history and use example config |
